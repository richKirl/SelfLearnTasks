# ROADMAP: GAME DEV (C++ / JAVA 25+) — ОТ 2D К 3D
==================================================

Cтруктурированный роадмап для обучения на
C++ 23/26 или Java 25/26

ЭТАП 1: 2D ОСНОВЫ (АРХИТЕКТУРА И ЛОГИКА)
---------------------------------------
1. Карточная игра (типа "Косынка") ✅ сделал С++23
   - Фокус: State Machine (состояния меню/игры), UI-события.
   - Hardcore: Реализация Undo/Redo через паттерн Command.
   - Стек: SFML/SDL2-3 (C++) / LWJGL (Java).(if Java 2D)

2. Платформер (вид сбоку)
   - Фокус: Гравитация, физика прыжка, коллизии с тайлами (AABB).
   - Hardcore: Парсер уровней из внешнего XML/JSON/TXT файла.

3. Idle-игра (Кликер/Менеджер)
   - Фокус: Работа с большими числами (BigInt), экономический баланс.
   - Hardcore: Бинарная сериализация сохранений (Save/Load).
   - Java 25: Использование Record Patterns для данных.

4. Шутер (вид сверху)
   - Фокус: Векторная математика (направление пули), Raycasting (линия видимости врага).
   - Hardcore: Система частиц (Particle System) для взрывов/гильз через Object Pooling.

ЭТАП 2: ИЗОМЕТРИЯ (2.5D — ТИПО D)
---------------------------------------
5. Изометрическая RPG (Action-RPG)
   - Архитектура: Переход на ECS (Entity Component System).
   - Математика: Проекция координат (Screen to World ISO).
   - Геймплей: Алгоритм поиска пути A* (Pathfinding) на сетке.
   - Графика: Алгоритм художника (Z-sorting) — чтобы персонаж заходил за объекты.

ЭТАП 3: ВХОД В 3D (НИЗКИЙ УРОВЕНЬ)
----------------------------------
6. Raycasting Engine (в стиле Wolfenstein 3D)
   - Суть: Псевдо-3D на основе 2D-карты.
   - Фокус: Работа с текстурными проекциями и углами обзора (FOV).

7. 3D Арена-шутер (в стиле Quake 1)
   - Технологии: Чистый OpenGL 4.5+ / Vulkan.(4.6)
   - Темы: Матрицы трансформации (Model-View-Projection), шейдеры (GLSL).
   - Загрузка моделей: Написание своего загрузчика .obj файлов.

ИНСТРУМЕНТАРИЙ
--------------------------------------
- Компиляторы: GCC 14+ / Clang 18+ .
- Сборка: CMake (стандарт индустрии).
- IDE: CLion, VS Code или IntelliJ IDEA (для Java).
- Отладка: GDB / Valgrind (поиск утечек памяти в C++).

ПРАВИЛО РОСТА:
"Каждая новая игра должна использовать 50% старого кода и 50% новых технологий."
==================================================

Краткий совет напоследок:
В C++ 23/26 обязательно используй import std; (модули), чтобы не ждать вечность при компиляции. В Java 25/26 используй Scoped Values и Virtual Threads, чтобы твои 10 000 монстров в "Д" не тормозили систему.
