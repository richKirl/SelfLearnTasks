## Задача: реализовать это, используя только документацию (cppreference) и свои знания, без генерации кода нейронкой.

Цель: «Жизнь» на Quadtree (Дереве квадрантов)
Вместо обычного массива используем Quadtree. Это позволит обсчитывать только «живые» области и потенциально бесконечное поле.

Этап 1: Модульная архитектура (C++20 Modules)
Разбейте проект на логические блоки. Никаких хедеров, только .cppm.

    module core; — основные типы (Point, Rect).
    module world; — логика Quadtree и обхода узлов.
    module render; — обертка над SDL2.
    main.cpp — точка сборки.

Этап 2: Дерево на стероидах (C++23/26)
Реализуйте узел дерева.

    Используйте std::expected (C++23): 
    
    Для функций поиска узла. Вместо возврата nullptr или исключений, 
    возвращайте expected<Node*, ErrorCode>. 
    Это заставит вас явно обрабатывать отсутствие узла.
    
    Используйте std::optional: 
    
    Для хранения состояния клетки.
    Умные указатели: Попробуйте обойтись без raw pointers, 
    только std::unique_ptr для дочерних узлов (NW, NE, SW, SE).

Этап 3: Алгоритм «Жизни»

    Реализуйте рекурсивный обход дерева.
    
    Условие: 
    
    Узел дробится (subdivide), если в нем есть хотя бы одна живая клетка, 
    и «схлопывается», если все клетки в нем умерли. Это сэкономит память.
    
    Магия: 
    
    Попробуйте написать функцию обновления состояния как const метод, 
    возвращающий новое дерево (иммутабельность — это сложно, 
    но очень круто для понимания владения памятью).

Этап 4: Визуализация (SDL2)

    Напишите простой рендерер, который рисует только заполненные 
    квадранты (границы узлов дерева). Это позволит увидеть, 
    как дерево «дышит», расширяясь и сужаясь вместе с колонией клеток.

Бонусный уровень (для C++26):(если есть 26 у меня 23)

Если ваш компилятор уже поддерживает зачатки Reflection или продвинутые 
static_assert с сообщениями, добавьте проверки на этапе компиляции, 
чтобы дерево не могло иметь глубину больше N.
